# Security Policy

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Instead, please report them via [GitHub Security Advisories](https://github.com/withplumix/plumix/security/advisories/new).

### What to include

- A description of the vulnerability
- Steps to reproduce
- Affected versions
- Any potential impact

### Response timeline

- **Acknowledgment:** within 3 working days
- **Initial assessment:** within 1 week
- **Fix timeline:** depends on severity, but we aim for patches within 2 weeks for critical issues

This project follows a **90-day disclosure timeline**.

> **Please do not report, discuss, or describe security issues on Discord, GitHub Issues, or any other public forum without prior contact and acknowledgment from the maintainers.**

### Scope

This policy covers the following packages:

- `plumix`
- `create-plumix-app`
- `@plumix/runtime-cloudflare`
- `@plumix/plugin-blog`
- `@plumix/plugin-pages`

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.x     | ✅ Latest minor only |

We only support the latest minor release. Upgrade to receive security fixes.
